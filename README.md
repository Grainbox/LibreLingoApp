# LibreLingo App

A language learning application built with Expo, React Native, and Expo Web. This is the frontend component of the LibreLingo project, consuming JSON exports from the [LibreLingo Python backend](https://github.com/librelingo/librelingo).

## Tech Stack

- **Framework:** Expo (v54)
- **Language:** TypeScript (strict mode)
- **Navigation:** Expo Router (file-based routing)
- **Animations:** React Native Reanimated
- **Gestures:** React Native Gesture Handler
- **Graphics:** React Native Skia
- **Storage:** WatermelonDB (offline-first local database)
- **Web Platform:** Expo Web (PWA)

## Architecture

### Platform Support

- **Mobile:** Android APK (via EAS Build)
- **Web:** PWA (Progressive Web App)
- **iOS:** Via Expo Go or EAS Build (future)

### Key Principles

- **Offline-first:** All course content and progress stored locally in WatermelonDB
- **Static JSON consumption:** Loads courses from JSON exports generated by the Python backend
- **Type-safe:** Full TypeScript strict mode
- **No external storage:** User progress never leaves the device in v1

## Project Structure

```
app/                          # Expo Router file-based routing
├── _layout.tsx              # Root layout with GestureHandler
└── (tabs)/                  # Tab navigation group
    ├── _layout.tsx          # Tab configuration
    ├── learn/
    │   └── index.tsx        # Course list and lessons (NAV-02, NAV-03)
    ├── profile/
    │   └── index.tsx        # User stats and settings (GAME-05)
    └── store/
        └── index.tsx        # Course marketplace (STORE-02)

src/                          # Source code organization
├── components/              # Reusable UI components
├── screens/                 # Full-screen components (not in routing)
├── services/                # Business logic services
│   ├── courseLoader.ts      # JSON loading and DB insertion
│   ├── answerValidator.ts   # Answer normalization and validation
│   └── syncService.ts       # Local-to-cloud sync (AUTH-03)
├── hooks/                   # Custom React hooks
├── utils/                   # Utility functions
├── types/                   # TypeScript type definitions
└── theme/                   # Design tokens (colors, typography, spacing)

assets/                       # Images, fonts, splash screens
└── courses/                 # Local course JSON files (populated at build time)

index.ts                      # Entry point (Expo App)
app.json                      # Expo configuration
tsconfig.json                 # TypeScript configuration
```

## Getting Started

### Prerequisites

- Node.js 18+ (recommend via nvm or volta)
- npm or yarn
- Expo CLI: `npm install -g eas-cli expo-cli`

### Installation

```bash
npm install
```

### Development

#### Web (easiest for development)

```bash
npm run web
```

Opens at http://localhost:19006 with hot reload.

#### Android (via Expo Go)

```bash
npm run android
```

#### Type checking

```bash
npm run type-check
```

#### Linting

```bash
npm run lint
npm run format  # auto-fix
```

## Course JSON Integration

Course data is loaded from static JSON files generated by the Python backend:

1. Place exported course JSON in `assets/courses/`
2. On app launch, `services/courseLoader.ts` imports them into WatermelonDB
3. All screens consume data from the local database

See the Python backend documentation for the [JSON contract](https://github.com/librelingo/librelingo/blob/main/dev-docs/json_contract.md).
