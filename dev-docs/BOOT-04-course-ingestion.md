# BOOT-04: Course JSON Ingestion Pipeline

**Status:** ✅ Implemented  
**Ticket:** BOOT-04 `P0` `[FRONTEND]`  
**Implementation:** Course loading pipeline with versioning by hash

---

## Overview

BOOT-04 implements the pipeline for ingesting course JSON files generated by the Python backend into the app's local database. It consists of two main components:

1. **`scripts/ingest-course.ts`** — CLI tool for processing course JSON files
2. **`src/services/courseLoader.ts`** — Service for loading courses into WatermelonDB

---

## Architecture

### Data Flow

```
Python Backend (courseData.json, skills/*.json)
         ↓
scripts/ingest-course.ts (hash + copy)
         ↓
assets/courses/<slug>_<hash>.json
         ↓
src/services/courseLoader.ts (parse + insert)
         ↓
WatermelonDB (local SQLite)
```

### Versioning Strategy

Courses are versioned by **content hash** (first 8 chars of SHA256):

- File format: `<courseSlug>_<versionHash>.json`
- Example: `spanish-a1_abc12345.json`
- When a course JSON changes, a new file is generated with a new hash
- The database tracks `versionHash` to detect updates

## Usage

### 1. Ingest a Course (One-Time)

The ingest script reads a course JSON from the Python backend and copies it to `assets/courses/`:

```bash
# Copy a course JSON and compute its hash
npx ts-node scripts/ingest-course.ts /path/to/spanish-a1.json

# Optionally specify custom output directory
npx ts-node scripts/ingest-course.ts /path/to/spanish-a1.json assets/courses
```

**Output:**
```
✓ Ingested course: spanish-a1
  Hash: abc12345
  Output: /path/to/assets/courses/spanish-a1_abc12345.json
```

### 2. Load Course Into App (Runtime)

In your app initialization code:

```typescript
import { initializeDatabase } from "@db/database";
import { loadCourseFromAsset, initializeDefaultCourses } from "@services/courseLoader";

// On first app launch
await initializeDatabase();
await initializeDefaultCourses();
```

### 3. Query Courses

```typescript
import {
  getAvailableCourses,
  getCourseBySlug,
  getSkillsForCourse,
  getChallengesForSkill,
} from "@services/courseLoader";

// Get all courses
const courses = await getAvailableCourses();

// Get a specific course by language code
const spanish = await getCourseBySlug("spanish");

// Get skills in a course
const skills = await getSkillsForCourse(spanish.id);

// Get challenges in a skill
const challenges = await getChallengesForSkill(skills[0].id);
```

## Database Schema

### Courses Table
- `slug` — Language code (indexed), e.g., `spanish-a1`
- `name` — Human-readable name, e.g., "Spanish A1"
- `sourceLanguage` — Source language code, e.g., `en`
- `targetLanguage` — Target language code, e.g., `es`
- `version` — Semantic version from course metadata
- `versionHash` — Content hash for change detection
- `imageUrl` — Course thumbnail (optional)
- `description` — Course description (optional)
- Timestamps: `createdAt`, `updatedAt`

### Skills Table
- `courseId` — Foreign key to courses
- `slug` — Skill identifier, e.g., `greeting`
- `name` — Display name, e.g., "Greetings"
- `description` — Summary of skill content (optional)
- `order` — Position in course (for sequential unlock)
- Timestamps: `createdAt`, `updatedAt`

### Challenges Table
- `skillId` — Foreign key to skills
- `courseId` — Foreign key to courses (denormalized for queries)
- `type` — Challenge type: `cards`, `shortInput`, `options`, `chips`, `listeningExercise`
- `priority` — Priority level (0, 1, or 2)
- `challengeData` — Full challenge JSON (stored as string)
- Timestamps: `createdAt`, `updatedAt`

## Course JSON Contract

The courseLoader expects course JSON to match this shape:

```typescript
interface CourseData {
  languageName: string;           // e.g., "Spanish"
  languageCode: string;           // e.g., "es"
  uiLanguage: string;             // e.g., "en"
  specialCharacters: string[];    // e.g., ["á", "é", "ñ"]
  repositoryURL: string;
  license: {
    name: { short: string; full: string };
    link: string;
  };
  modules: Array<{
    title: string;
    skills: Array<{
      id: string;                 // e.g., "greetings"
      title: string;
      summary: string[];
      levels: number;
      practiceHref: string;       // Path to skill JSON (e.g., "skills/greetings.json")
      introduction?: string;
      imageSet?: string[];
    }>;
  }>;
}
```

## Implementation Details

### Hash Computation

The ingest script computes SHA256 hashes:

```typescript
function computeHash(content: string): string {
  return crypto
    .createHash("sha256")
    .update(content)
    .digest("hex")
    .substring(0, 8); // First 8 chars
}
```

### Update Detection

When loading a course:

1. Check if course with same slug exists
2. Compare `versionHash` in database vs. new JSON
3. If different, delete old skills/challenges and reload
4. If same, skip (course already up-to-date)

### Transaction Safety

All database operations are wrapped in `db.write()` transactions to ensure consistency.
